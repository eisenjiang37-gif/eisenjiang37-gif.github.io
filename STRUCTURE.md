## Pages

### `research.html`

**Tabs** — driven by `data-tab` / `id` attributes and `main.js` tab logic.

**Paper / thesis entries** — markup pattern:
```html
<div class="item">
  <div>
    <div class="item-title">…</div>
    <div class="item-authors">…</div>
    <div class="item-venue">…</div>
    <div class="item-links">
      <a href="…" class="item-link">PDF</a>
      <button class="item-link bib-toggle" data-bib="bib-<id>">BibTeX</button>
    </div>
    <div class="bib-block" id="bib-<id>"><pre>@…</pre></div>
  </div>
  <div class="item-year">YYYY</div>
</div>
```

`data-bib` on the button must match the `id` of its `.bib-block`.
Add the corresponding entry to `bib.tex` when adding a paper.

### `writing.html` — Blog

The blog is a **single-page reader**: one HTML shell, posts loaded dynamically from a compiled manifest.

**Data flow:**

```
blogs/*.md
    ↓  gen_index.py  (run manually)
blogs/index.json   — fetched at runtime by blog.js
blogs/index.js     — same data as a JS variable (fallback / preload)
    ↓  blogs/blog.js
writing.html #postList  +  #blogContent
```

**`gen_index.py`** (run after adding/editing posts):
- Scans `blogs/*.md`, compiles each to HTML with the built-in `md_to_html()` converter
- Extracts title from first `# H1` (falls back to filename)
- Extracts optional tag from `<!-- tag: Note -->` comment in first 20 lines
- Uses file **modification time** as post date
- Writes `blogs/index.json` and `blogs/index.js`
- Rewrites `README.md` with the current post list

**`blogs/blog.js`** — runtime:
- `loadManifest()` — uses `window.__BLOGS_INDEX__` if present (from `index.js`), otherwise fetches `index.json`
- `init()` — renders the post list in `#postList`; loads post from URL hash (or first post)
- `loadPost(post)` — injects pre-compiled HTML into `#blogContent`
- `setListVisible()` — toggles `.list-hidden` on `.blog-layout`; persisted in `localStorage("blog-list-hidden")`
- `#listToggle` / `#listExpand` — collapse/expand the post list panel

**To add a blog post:**
1. Drop a `.md` file into `blogs/`
2. Run `python3 gen_index.py`
3. Commit `blogs/index.json`, `blogs/index.js`, `README.md`

## Theme system

Switching is handled entirely by `main.js` (lines 4–25) + CSS variable overrides.

- Default theme: `dark` (set in `<head>` inline script on each page to prevent flash)
- Toggle persists to `localStorage("theme")`
- CSS: `:root` = dark vars; `html[data-theme="light"]` = light overrides
- To change a color: edit the variable in `:root` / `html[data-theme="light"]` in `style.css`

---

## Assets convention

All static files served from `assets/`:
- `style.css` — styles
- `main.js` — scripts
- `*.pdf`, etc. — downloadable attachments

Link with a root-relative path from any page: `assets/<filename>`.
